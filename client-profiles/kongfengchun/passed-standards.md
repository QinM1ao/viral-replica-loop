# Passed Standards

这些是孔凤春项目里已经通过的标准。后续 loop 必须用这些作为通过线。

## Image Gen 分镜通过线

- Codex built-in ImageGen must have a passing `codex_imagegen_contract_qc.json` for image sample and image batch.
- The ImageGen contract must prove actual source storyboard, product front, open-white-mud, and identity images were loaded as references.
- Source storyboard can only provide layout, shot order, framing, and action rhythm. It cannot provide old product, old tool, old source host identity, old mud color, or subtitles.
- 12 格分镜必须保持源视频画幅；竖屏源片默认 4 列 x 3 行，横屏源片默认 3 列 x 4 行，不能被改成长条或挤压。
- Shot 顺序必须跟通过的镜头表一致。
- 单男模，全程同一个人；不能出现第二个男模。
- 人物脸、肤色、曝光要贴近用户给的人物参考，不要贴近原视频主播。
- Part1 / Part2 肤色和光线差异不能明显。
- Part1 / Part2 必须像同一条视频的连续切片；人物、发型、衣服体系、场景、光线、泥膜质感不能明显割裂。
- 默认完整换人：脸、发型、身体和衣服都跟模特图；源视频衣服属于旧人物外观，必须替换掉。
- 对 `job-003` 当前女模，白色小背心是目标服装；黑色源视频服装不能作为最终交付服装。
- 产品必须是孔凤春白色方圆罐，有绿色标识。
- 不能出现陌生包装盒、礼盒、纸盒、说明盒、白罐替身。
- 泥膜必须是乳白/奶白厚泥；Shot06 / Shot10 这类近景要能看出厚膏体和小峰。偏黄、米色、肤色、暖奶油色或灰色都不算通过。
- 白泥不能只靠颜色过关。脸上和罐内都要有凸起、堆叠、边缘、纹理或小峰的厚泥膏体感；薄薄一层像乳液、粉底、透明霜或平刷白色，即使机器白度 PASS 也不能通过。
- 使用前/洗前脸不能已经像洗后一样白净发亮。必须能看出轻微油光、毛孔、闭口、暗沉、粗糙或正常未清洁肤感，洗后改善只能发生在洗脸/擦净之后。
- 上脸动作是手指/指腹涂抹，不是管状或棒状涂抹头。
- 旧片中的管状/棒状/刷头/手臂试色动作必须先改写成手指从罐内取白泥、指腹上脸；通过稿里不能出现泥膜棒、涂抹棒、刷头、棉签、手臂试色。
- 不把旧片灰泥、手臂试色、管状工具带进新分镜。

## Seedance 提示词通过线

- 提示词必须是干净的提交稿，不写内部失败史。
- 提示词必须按当前 `references/seedance-20-prompt-standard.md` 结构写：`参考图角色` + 精简全局规则 + `秒数｜Shot 01–02` 执行块，每块就地绑定 `画面 / 声音`；有真实动作音时才写 `音效`。
- 局部问题只能做锁块修复：标记唯一失败的 `时间｜Shot` 块，只改该块，允许范围外的提示词块和已通过 Part 必须保持不变。不能借“简化”重写整段。
- 上脸涂抹等物理动作必须写清接触位置、指腹运动和留下白泥的结果，不能只写“涂”。合并块内需要定位动作时继续保留具体子 Shot 编号，不改成“局部画面/另一侧”等泛称。
- 镜头执行块和说话组必须分开；单块最多 5 格。场景、画面功能或声音模式变化时必须新开执行块，画面内同期口播和画外旁白不得混在同一块。手机 proof、产品名特写、洗后证明、结尾产品 close-up 等源片关键 beat 必须复刻，不得再把 Shot 02–10 压成一个大块。
- 短促静物 B-roll 必须独立成块。若原片是主播产品近景 → B-roll → 主播产品近景，B-roll 前后的场景、景别和产品位置要匹配，不重新起手推近，不擅自改成溶解转场。
- 最终提示词只能写模型可理解的素材角色，不能写 `Part1最终分镜图`、`AI改好分镜图`、`current-job`、`source rhythm board` 等内部工作流词。
- 口播必须打中文引号，例如：“孔凤春清洁泥膜”。
- 整条成片只说一次完整产品名 `孔凤春清洁泥膜`，在产品特写镜头上独立成句；其他 Part 用“这罐泥膜”，不重复品牌名，不写“孔凤春泥膜”等自造缩写。
- 图片引用要写清角色：
  - 分镜图控制镜头/动作/节奏。
  - 人物图控制脸和身份。
  - 产品正面图控制罐身/标签。
  - 开盖图控制白色厚泥。
  - 身份图只锁同一人物身份和服装，不传递参考图背景，也不覆盖洗前/洗后皮肤状态。
  - 当前 profile 默认不传独立洗后亮肤图；只有规则显式要求时才增加。
- 网页端手动 Seedance 交付必须有一个最终输出区，集中放上传素材、提示词、manifest、音频和说明。
- Part2 开始必须直接有运动和环境动作，不能从静止头像停住。
- 多段视频默认无 BGM，只保留口播、环境声、操作声。
- 洗后结果必须达到当前 profile 的明确商业效果：干净透亮、平滑细腻，黑头、闭口、粉刺、油光和粗大毛孔消失，近距离呈现完全无瑕疵；不得写“保留真实毛孔/皮肤纹理”主动降级。
- 当前首选实测标准是 `references/kongfengchun-final-26s-standard.md` 及其 Part1/Part2 标准稿；旧 Part2-only 和发酵水案例只作局部经验或历史结构参考。

## 最终视频通过线

- 产品镜头、产品名、字幕/口播节奏要对上。
- 两段接缝不能冻结、不能突兀停顿。
- 前后肤色不能像换了灯、换了人。
- 洗后变好必须发生在洗脸后，不提前出现。
- 洗前不能提前套用洗后亮肤参考；如果洗前已经太白太干净，前后对比不成立。
- 泥膜颜色和质感前后一致。
