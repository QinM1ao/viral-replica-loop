# Seedance 2.0 复刻提示词标准

这是项目内 15 秒 Seedance 2.0 复刻提示词的唯一格式事实源。

## 当前任务人物性别

- 先读取当前 job 的角色映射、`approved_visual_manifest.json` 和身份组 manifest。
- `source_presenter_gender`、`target_presenter_gender`、身份组 `presenter_gender`、导演计划 `presenter_gender.source/target` 必须一致。
- 男主播任务只使用“男主播/男声”，女主播任务只使用“女主播/女声”；示例人物不得进入当前任务。
- 允许配角保留原片性别，也允许“女朋友”等剧情台词；门禁针对主角、主播、主持人和主声音标签。
- 最终 Prompt 必须来自 renderer 并通过 gender contract QC，不能把聊天手写草稿当最终文件。

当前首选实测标准是 2026-07-17 `job-011` 的完整 26 秒双 Part 案例：

- 完整案例入口：`references/kongfengchun-final-26s-standard.md`

## 核心结论

内部导演计划必须把 `beats`、`speech_groups` 和 `execution_blocks` 分开。`speech_groups` 只负责台词与发声容量，`execution_blocks` 负责镜头、场景、画面功能和声音模式都能一起执行的最小分块。交给 Seedance 的最终提示词在每个执行块同时写清画面和声音；音效是可选行，只在画面确有有用的对应动作音时出现。

提示词局部修复必须使用**锁块修复**：只改用户或 QC 指定的失败 `时间｜Shot` 块，其他执行块、参考素材、模型、时长和已通过 Part 全部保持不动。修改前后必须做区块 diff；允许范围外出现任何措辞变化，都不算局部修复完成。

已在当前批准分镜中完成的删除不进入最终 Prompt。若字幕、贴纸、旧产品排或其他对象已从 `@图片1` 像素中清除，模型面对的画面指令只描述分镜里实际保留的人物、动作、场景和产品镜头；不再写“前景无产品”“不陈列产品”“不出现旧产品”等删除说明，也不在 `scene_rule` 重复总结。删除依据保留在内部 `visual_edits` 和检查证据中，不向 Seedance 暴露被删对象的名称。

`source_rhythm.json`、原视频、ASR 和人工分析只用于在上游推导 Shot 动作，不是 Seedance 实际接收的素材。最终 Prompt 只能引用实际上传的 `@图片 / @音频` 和已展开的具体动作。不得出现“原片 / 源片 / 原视频 / 按分镜 / source video / source rhythm / source beat”。

最终格式：

```text
0.0–2.8秒｜Shot 01–02
画面：[引用已上传@图片的具体Shot，写已展开的目标动作]
声音：[说话人模式]{[本块台词]}。
音效：<[画面里实际发生且值得强调的动作声]>
```

符号是硬格式：口播只能写在 `{...}` 中；音效必须逐项写成 `<...>`，多个音效用 `；` 分隔。中文引号台词、未包裹尖括号的音效、`音效：无` 都不能通过最终 Prompt QC。没有音效时整行省略。

## 原台词锁定

台词默认不是“按原片功能改写”，而是 `source_locked`：

- `source_rhythm.json` 中经证据确认的台词按原用词、句序、重复、说话人、重音和停顿保留。
- 只修改用户或当前产品事实要求变更的局部槽位；每处修改都要登记在 `speech_group.line_edits`。
- 只保留原句的“功能、意思或剧情节点”，然后重写成另一套电商文案，必须 `FAIL`。
- 目标时长更短时，先保留原片语速与原句骨架；只能用已声明的 `duration_compression` 删除重复/填充片段。
- 最终 prompt 中每个 `声音：` 由 renderer 机械继承 `speech_group.line`；不在 prompt 阶段二次润色。

完整可改槽位、`line_edits` 格式和门禁见 `source-script-lock.md`；台词和镜头共同服从 `source-replication-contract.md` 的 `necessary_only` 总合同。

禁止回退到：

- 三个粗粒度 `Shot 1 / Shot 2 / Shot 3` 大段
- 画面写完后另立一个“声音执行”区
- 每个分镜重复一遍同一句台词
- 只写秒数，不写分镜图上的 Shot 编号

## 生成前的事实映射

最终提示词不能只看分镜图发挥。先在 `director_plan.json` 和 `voiceover/shot_line_map.md` 中完成：

```text
目标时间 -> Shot 范围 -> 原片时间 -> 原片画面动作
-> 原片说话人/台词 -> 目标画面动作 -> 目标说话人/台词
-> 音效 -> 参考素材绑定 -> must-keep 原因
```

目标动作只有两种合法来源：

1. 原片/分镜明确发生的动作。
2. `product_profile.json` 要求的必要产品动作翻译。

不得为了让过程“更完整”而补写原片没有的起手动作。原片从已打开的包里取瓶，目标就是“从已经打开的包中直接拿出产品”，不能改成“打开包、拉开拉链、再拿瓶”。

原片是硬切时，提示词明写“硬切”。不让模型把两个独立镜头补成一个连续变形动作。

## 分镜分块

- 标题直接使用 `秒数｜Shot 编号范围`。
- Shot 编号必须对应 `@图片1` 上保留的 `Shot 01–12`。
- 标题是本块的主 Shot 定位。合并块内存在不同动作、说话落点或产品落点时，正文继续明确写 `Shot 03`、`Shot 04` 等子 Shot；不要为了“简洁”改成“局部画面、另一侧、正脸近景”等无法唯一寻址的泛称。只有单一动作块才省略正文里的重复编号。
- 相邻 Shot 画面、场景、声音归属和动作功能一致时，可合并成 `Shot 01–02`。
- 不为了凑数量固定拆成 3 个块或 6 个块；通常 3–8 个执行块。
- “3 个块”只是稀疏内容的最低门槛，不是默认目标。有逐拍原片证据的密集口播广告通常需要 5–8 个音画块。
- 单个执行块最多覆盖 5 个 Shot 面板；这是安全上限，不是尽量塞满的合并目标。原片功能或台词证据落点已变（例如产品出现 → 成分解释 → 使用指令，或洗后证明 → 手机打卡 → 空罐证明）时必须分块。
- 场景、画面功能或声音模式任一发生变化，都必须新开执行块。画面内同期口播和画外旁白不得挤进同一个执行块。
- 短促静物 B-roll 是独立画面功能，必须单独成块；不能与前后的主播产品镜头合并。
- 原片是 `主播/产品近景 → B-roll → 主播/产品近景` 的硬切时，B-roll 前一块结束就让产品到达目标近景，后一块开镜直接保持同一场景、相近景别和产品位置。不要重新起手推近，也不要擅自加溶解转场。
- 一个说话组不得跨越两个执行块；需要分块时，先按原片短句重新切分说话组，不把一整句重复到两块。
- 每个 Shot 01–12 都必须被某个块覆盖，不跳过、不调换。
- 多个连续画面合并时，用“硬切到”、“随后”等真实剪辑关系连接，不把动作堆成同一时刻并发。
- 物理变化镜头必须写出“接触对象/位置 → 可见运动 → 结果状态”。例如涂泥要写指腹接触鼻翼、真实划过并留下白泥；只写“涂”不足以约束真正上脸。

## 局部提示词修复

生成视频只有局部镜头失败时：

1. 先保存当前通过稿为 baseline，并标记唯一允许修改的 `时间｜Shot` 块。
2. 只修失败原因。动作缺失就补接触、运动和结果；场景泄漏就修该块的素材角色或场景句；不要顺手缩写、重排或润色其他块。
3. 重新渲染后比较执行块。允许范围外必须与 baseline 完全一致。
4. 只重跑失败 Part。其他已通过 Part 不重写、不重新提交。
5. 用户确认新版 Part 后，把它和其他已通过 Part 锁定，再进入拼接与最终 QC。

`references/kongfengchun-final-26s-standard.md` 是已通过案例：Part1 只改 `0.0–2.2秒｜Shot 01–02`，`2.2–13.0秒` 保持原有明确 Shot 写法。此前把后半段一起“简化”的版本导致景别和镜头执行漂移，属于错误流程。

## 声音和口播

- 台词写在它所属的 Shot 块内，只出现一次。
- 每句继承原片说话方式：原片是画外音，目标也是画外音；原片是画面内同期口播，目标也是画面内同期口播。
- 不用“只发声不上屏”。全片统一说“无字幕”就够了。
- 不生成任何背景音乐。只保留指定台词、真实环境声和与可见动作同步的音效。
- 音效必须由画面中的真实动作产生。不能先编一个音效，再反向给画面增加动作。
- 15 秒口播容量以 `voiceover-capacity-and-compression.md` 为唯一数值事实源。完整复刻容量不足时优先调整 Part 边界或允许成片在技术容差内略长，不得删改完整台词。只有用户明确要求短版，才能按 `user_compressed` 删除有证据标记的重复/填充内容。
- 无逐拍源片证据时最多 3 个说话组；有 `source_rhythm.json` 的原片短促发声 beat 可按证据拆到最多 6 组，但不增加总发声单位。

## 产品名发音锚点

- 产品名的提及次数、位置、句式和重复默认继承原片；每个旧产品名只在原位置局部替换为 job `product_name`。
- 原片没有念产品名时，`spoken_product_anchor={"enabled": false}`，不得为“品牌露出”新造一句口播。
- 原片念了产品名时，`spoken_product_anchor.full_name` 必须与 job `product_name` 完全一致，并绑定到其中一个含 `@图片2` 的原有产品证明镜头；它只负责音画落点，不控制全片提及次数。
- 原片把产品名嵌在钩子或长句中，目标也保留原句结构；原片独立成句时才独立成句。
- 原片重复品牌名时保留相同次数和位置，不自动改成“这罐泥膜/这瓶水/这件产品”；只有原片本来使用代词或用户明确要求时才使用。
- 不写未批准的品牌缩写。

## 参考素材绑定

```text
参考图角色：
@图片1定义为“分镜板”，只控制镜头顺序、场景、景别、动作和剪辑节奏；不传递分镜网格、边框、编号、画面文字、旧产品及旧人物身份。
@图片2中的产品定义为“目标产品”，只锁定包装、标签、比例和材质；不传递白底或棚拍背景，不控制构图和动作。
@图片3中的产品定义为“目标产品材质”，只锁定[开盖/瓶口/质地]；不传递参考图背景和构图。
@图片4中的人物定义为“主角”，只锁定脸、发型、身体、服装和身份一致性；不传递人物参考图背景、构图和其他角色身份，也不覆盖各 Shot 指定的使用前/洗后皮肤状态。
@音频1定义为“声音表演参考”，只锁定音色、自然停顿和语速；台词完全按当前 source-locked 计划执行，不增删或改写。
```

这个“定义 → 只锁定/控制 → 不传递”开头是标准版。“控制校准……”、“综合参考所有图……”等模糊句不是最终提示词格式。无模特多人分支的角色A/B 和每 Part 素材映射见 `storyboard-derived-identities.md`。

最终提示词不写仓库内部名称：`Part1最终分镜图`、`AI改好分镜图`、`current-job`、`source rhythm board`、`contact sheet`、`素材角色表`。

## 标准开头

```text
生成约{duration_seconds}秒、9:16竖屏、720p真实手机短视频。按@图片1的Shot顺序和硬切节奏执行；保持同一主播和同一服装，场景严格按对应Shot执行，不继承人物参考图背景。每镜完成动作后切换，不生成冻结帧或慢动作。全片无字幕，不生成任何背景音乐，只保留口播、环境声和同步动作音效。
```

画幅、分辨率、产品标签和人物信息按当前任务替换。每个 Part 都是独立 Seedance 任务，必须在本段重新完整绑定素材；不写“同 Part1”或“承接上一段”。

## 错误示例：不得脑补动作

错误：

```text
画面：女主打开黑色手提包，拉开拉链，再从包中拿出产品。
音效：<拉链声>；<取出瓶身的摩擦声>
```

原片从“包已打开”开始，错误示例多加了拉链步骤。它会迫使模型在短时间内同时完成开包、拿瓶和产品特写，容易出现手部重叠、动作未完成就进入下一镜和物体变形。

正确：

```text
5.9–8.2秒｜Shot 05–06
画面：硬切到已经打开的黑色手提包，从包中直接拿出目标产品；随后硬切到真实场景中的手持瓶身近景。
声音：旁白{[对应台词]}。
音效：<瓶身从包内取出的轻微摩擦声>
```

## 错误示例：说话组不能把画面压成大块

错误：

```text
2.0–10.7秒｜Shot 02–10
画面：洗后脸证明、手机第三十天、空罐、产品推镜全部挤在一块。
声音：主播同期口播{重复品牌名的一整段台词。}。
```

这会让模型同时追踪多个功能、道具和人脸状态，容易冻结、镜头重叠、口播错位和产品名发音失真。正确做法是分开 `speech_groups` 与 `execution_blocks`，按洗后证明、手机/空罐证明、护肤结论分块，单块最多 5 格。

## 错误示例：不能用“真实感”削弱商业效果

错误：

```text
洗后油光减少，保留真实毛孔和皮肤纹理。
```

当 brief 或产品 profile 要求明显的商业化使用后效果时，这句话会把目标主动降级。应直接写清需要消失的问题和最终状态；孔凤春清洁泥膜的当前标准见 `references/examples/seedance-20-kongfengchun-part2-validated.txt`。身份参考只锁人物，皮肤结果由对应 Shot 控制。

## 错误示例：B-roll 不能和主播收口混成一块

错误：

```text
8.9–11.8秒｜Shot 09–11
画面：主播把产品推近，然后切多个瓶罐。
声音：主播同期口播{前半句。}；旁白{后半句。}。
```

这里同时跨了主播镜头、静物 B-roll 和声音模式。正确做法是拆成 `Shot 09–10`、`Shot 11`、`Shot 12` 三块；Shot 10 与 Shot 12 使用匹配的场景和产品近景，让原片硬切自然成立。

## 错误示例：局部问题不能重写全段

错误：

```text
开头涂抹不真实，于是把 Part1 后面所有 Shot 一并压成“继续涂抹、展示产品、指出位置、举手”。
```

这会让已经通过的景别、动作、台词落点和产品镜头重新变成未约束状态。正确做法是只补开头失败块的“接触 → 运动 → 留泥结果”，后续通过块逐字锁定。

## 完成条件

提示词只有同时满足以下条件才算完成：

1. 参考素材角色完整，`@图片1` 明确是带 Shot 编号的当前分镜。
2. 全片无字幕，不生成背景音乐。
3. 每个执行块都有 `画面 / 声音`；有真实动作音才写 `音效：<...>`，没有时整行省略，不写 `音效：无`。
4. 执行块使用 `时间｜Shot 范围`，按顺序覆盖全部 Shot。
5. `director_plan.json` 存在独立 `execution_blocks`，单块最多 5 格，没有说话组跨块，也没有一个执行块跨同期口播/画外旁白边界。
6. 没有独立“声音执行”区，没有额外台词，每句台词只出现一次。
7. 产品名的出现次数、位置、句式和重复与原片一致；原片不说产品名时禁用锚点且不新增口播，原片说了才在原位置局部替换并绑定证明落点。
8. 每个目标动作都能回指到原片动作或明确的 product-profile 翻译；没有自行补全过程。
9. 每个音效都对应当前画面里真实发生的动作。
10. 口播通过 `voiceover-capacity-and-compression.md` 的容量检查，说话人模式与原片一致。
11. 多场景 Part 有明确的 Shot 场景图；人物参考图背景不会覆盖分镜场景。B-roll 前后若按原片硬切，入口与出口景别已匹配。
12. 护肤效果强度符合 brief/profile，没有擅自追加会削弱目标的“保留真实毛孔/纹理”等句子。
13. `python3 tools/seedance_prompt_contract_qc.py --root . --job-id <job-id>` 通过。
14. 独立 checker 对照原片视频、ASR、分镜和 `shot_line_map.md` 确认无脑补动作后才能 PASS。
15. 局部修复有 baseline、唯一允许修改块和区块 diff；允许范围外没有任何变化，已通过 Part 没有重跑。
16. 每个 `physical_change` 镜头都明确写出接触、运动和可见结果，不用单独一个“涂/洗/擦”动词代替动作。
17. `director_plan.json` 使用 version 6、`script_fidelity.mode=source_locked` 和 `replication_fidelity.change_policy=necessary_only`；`source_script_fidelity.md` 与 `source_replication_fidelity.md` 证明每句、每个镜头只有已登记的必要局部修改；最终 prompt 与目标计划逐句逐镜头一致。

## 哈希不是 Seed

SHA-256 哈希只是文件内容指纹，不控制 Seedance 随机性，也不能复现或改善视频。项目内部保留它，用来识别素材是否换版、避免旧 QC 错配、复用未变化素材的检查，以及核对请求、提示词和成片关系；日常用户流程不展示哈希，也不要求用户管理哈希。Seed 才是模型随机生成参数，两者没有替代关系。
