# Seedance 2.0 提示词参考

本文档保留跨品类素材绑定说明和历史案例。最终 Seedance 2.0 提示词格式与当前首选实测案例只读 `seedance-20-prompt-standard.md`，不要从本文件复制旧案例格式。

## 基本约束

- **比例：** 默认按源视频/用户 brief 的目标画幅；只有明确要求竖屏投放时才写 9:16 竖屏
- **时长：** 单次最长 15 秒
- **输入：** 文字描述 + 参考图（改好的分镜网格图）
- **生成：** 用户要最终视频时，直接调用 Seedance 生成 Part、下载、ffprobe 验证，不要只交付提示词。
- **多段声音：** 30秒双段/多段拼接默认禁止BGM，只保留对白、环境声、真实操作音。单段氛围片可按用户要求带BGM。

## 生成前素材角色表

写提示词前，先输出 `seedance_素材角色表.md`。不要跳过这一步。

模板：

```markdown
# Seedance 素材角色表

## Part 1

生成模式：参考图生视频 / 真人素材库 / 参考音频 / 纯文字避污染
主目标：产品还原 / 人物身份 / 动作节奏 / 场景氛围 / 口播声音
次目标：
主动简化：

素材绑定：
- 图片1：最终改图分镜，可以保留源 Shot 编号。传递：镜头顺序、构图、手部动作，以及 Shot 编号与 panel 的寻址对应关系。禁止传递到最终视频：黑底排版、分镜边框、Shot编号；旧字幕、旧产品不得存在于已批准改图分镜中。
- 图片2：产品参考。传递：品牌/包装/色号/材质/可见形态。禁止传递：拍摄背景和无关道具。
- 图片3/图片4：按 product profile 的实际顺序放材质或人物身份。人物身份传递脸、发型、妆容、服装和年龄感；禁止传递参考图背景、构图和 Shot 皮肤状态。
- 音频1：声音参考。传递：语速、停顿、直播感。禁止传递：旧台词、旧产品名、BGM。
```

每段只选一个主目标、一个次目标。主目标是验收核心，次目标能后期修就不要重跑。

## 提示词格式规范

每个 Part 必须输出为一个可直接复制到 Seedance 的独立 `text` 代码块。不要只交付“通用要求 + 摘要说明”，也不要让用户自行拼装。结构如下：

1. **素材绑定声明**：图片1/图片2/图片3/音频1分别控制什么，并写清不传递什么。
2. **开头声明**：根据我提供的第X张分镜图和用户 brief，生成一条15秒真实短视频；画幅按源视频/brief，只有明确要求竖屏时才写 9:16。
3. **人物设定**：具体到年龄/发型/妆容/穿着，写一整句
4. **产品设定**：具体到颜色/质感/形态/包装，写一整句
5. **场景设定**：具体到背景/灯光/桌面/氛围
6. **视频要求**：用列表写禁止项和约束条件
7. **音画执行块**：给模型的最终提示词用 `秒数｜Shot 01–02` 定位每个关键段，每块就地写 `画面 / 声音`；有真实动作音才增加 `音效：<...>`，没有就省略整行。
8. **台词唯一性**：内部 `speech_group` 可以覆盖多个连续分镜，最终提示词将这些镜头合并成一个执行块，台词在该块只出现一次。组数和容量按 `voiceover-capacity-and-compression.md` 执行。

### 关键写法要点

- 每个 Part 都是独立 Seedance 任务。每段都要完整写人物、产品、场景和 @图片绑定，不能写“同Part1完全一致”“后续继承同一个人”这种依赖上下文的句子。
- 不写“参考所有图片风格”。每张素材都必须有主角色：分镜、产品、人物、环境、首帧、尾帧、声音节奏、动作节奏。
- 图片引用必须用 Seedance 的 @图片 写法。产品特写、开盖膏体、包装靠近镜头时，在对应时间段再次 @ 产品图；人物身份在每个独立 Part 的人物设定里 @ 一次。洗后皮肤参考只有 product profile 明确要求时才上传，并且只在洗净/擦净后的时间段 @。
- 画面描述要极具体：不写"展示产品"，写"右手拿起一根鸭脖举至脸旁，目光看向镜头"
- 带 reference audio 时，台词仍写在对应 Shot 执行块内，只出现一次；不创建独立“声音执行”区。
- 禁止项要详细列举：字幕、标题条、说明文字、贴纸、水印、logo、分镜边框、黑底排版
- "成片是正常短视频，不要保留分镜图的黑底排版"——这句很重要，防止 Seedance 照搬网格排版
- 如果有多段，每个 Part 都按独立 Seedance 任务写：前一段完成自己的最后动作和台词；后一段按原片对应镜头独立开始，并在本段内重新绑定人物、产品、场景和声音参考。原片是硬切或时间跳跃就照原片切，不继承上一段最后一帧。
- 不要在提示词里写内部纠错说明或污染词，例如“不是孤立商品图”“产品不是泥膜棒”“不要裁头”“不拆衣服”“方便拼接”“静止余量”“焦点锁产品”。改成正向事实：白色圆罐、开盖乳白泥膏、头肩上半身完整入画、黑色无袖上衣、手部动作连续。
- 食品产品必须写可见形态，不要只写味道。比如鸭脖要写骨节、骨孔、骨头截面、脊骨纹理、肉丝纤维、调料颗粒，并列出禁止误生成项。
- 用户明确品牌/产品后，提示词、口播、文件名都必须锁定该说法；不要沿用参考视频标题、字幕或模型识别出的旧品牌名。

### Seedance 2.0 编号分镜复刻格式

15秒网页端或 R2V handoff 使用 `references/seedance-20-prompt-standard.md` 的编号分镜格式。按原片功能把 Shot 01–12 合并成通常 3–8 个音画执行块，不固定凑成三个大段。

最终提示词必须写给视频模型看，不能写 loop 内部叫法：

- 用 `@图片1 控制镜头顺序、景别、动作节奏`，不要写 `Part1最终分镜图`。
- 用 `@图片2 控制产品正面包装`，`@图片3 控制开盖白泥质感`。
- 用 `@图片4 控制人物身份`；它不传递参考图背景，也不覆盖各 Shot 的皮肤状态。只有 product profile 明确要求时才增加 `@图片5` 洗后参考。
- 用 `秒数｜Shot 01–02` 写动作、口播和音效；每句口播必须落在原视频对应镜头功能上。
- 每个目标动作必须有原片证据或明确的产品动作翻译；原片是硬切就保留硬切，不补写原片没有的起手步骤。
- 场景、画面功能或声音模式变化时必须新开执行块；同期口播与旁白不能混块，短促静物 B-roll 单独成块。

复刻任务里的关键镜头不能被合并吞掉。原片有手机举给观众看、产品名落在产品特写、洗脸后证明、结尾产品 close-up 时，必须在对应 Shot 执行块里明确写出。

## 30秒压缩复刻

当用户说“长的缩到30秒”“只做一个30秒小剧情”“复刻整体节奏和内容”时：

1. 不按原视频全长拆分。
2. 固定生成两个15秒Part。
3. 从原片均匀抽24帧，分2组，作为整体节奏参考。
4. Part1承接原片开场钩子、冲突、问题放大、核心卖点引入。
5. Part2承接原片证明点、解决方案、行动收口。
6. 每段都写清楚“禁止BGM、禁止配乐、禁止音乐铺底、禁止鼓点”。
7. 把“音乐卡点”改成“动作卡点 / 灯光卡点 / 台词停顿点 / 操作音卡点”。

示例声音约束：

```text
声音：禁止BGM，禁止配乐，禁止音乐铺底，禁止鼓点。只保留旁白、自然环境声、面板点击声、灯光开启声、窗帘滑动声等真实操作音。开头和结尾保留0.3-0.5秒自然环境声，方便和相邻Part拼接。
```

如果用户后期要统一配音/配乐，生成后额外导出静音保险版：

```bash
ffmpeg -y -i "<最终30秒.mp4>" -c:v copy -an "<最终30秒_静音保险版.mp4>"
```

## 多段音频边界

带 reference audio 的多段视频，生成前必须按 ASR 句子边界切音频。不要机械 `0-15s / 15-30s`：

```bash
# 例：最后一句 14.56s 结束，补 0.44s 静音到 15s
ffmpeg -y -i "<原片音频.wav>" \
  -filter_complex "[0:a]atrim=0:14.56,asetpts=PTS-STARTPTS[a0];anullsrc=channel_layout=mono:sample_rate=16000,atrim=0:0.44,asetpts=PTS-STARTPTS[s];[a0][s]concat=n=2:v=0:a=1[a]" \
  -map "[a]" -ar 44100 -ac 2 -c:a libmp3lame -q:a 3 "<out>/voice_part1_clean.mp3"
```

提交 Seedance 前转写每条 reference audio，确认：

- Part1 没有 Part2 开头句。
- Part2 没有 Part1 尾句。
- 提示词音频脚本和实际 reference audio 一致。
- 最后 0.3-0.6 秒是环境声/动作声余量，而不是下一段开头。

## Seedance 直接生成

### 本地参考图生成

本地图片不能直接传给 Seedance。用 `run_seedance_magic_mirror.py --image-file` 上传到 `api.qinmiao.space` 后再生成：

```bash
source "$HOME/.config/wujieai/env"
python3 ~/.codex/skills/seedance-magic-mirror-video-prompt/scripts/run_seedance_magic_mirror.py \
  --prompt-file "<Part提示词.txt>" \
  --image-file "<分镜或参考图.jpg>" \
  --output "<输出Part.mp4>" \
  --model ep-20260521101914-nwv8j \
  --ratio 9:16 \
  --duration 15 \
  --resolution 720p \
  --generate-audio \
  --image-role reference_image \
  --poll-interval 10 \
  --max-wait 5400
```

### 纯文字生成

参考图含旧品牌、医疗感、老人输液、敏感或无关元素时，改用纯文字生成，避免污染：

```bash
source "$HOME/.config/wujieai/env"
env -i PATH="$PATH" HOME="$HOME" GATEWAY_API_KEY="$GATEWAY_API_KEY" \
python3 ~/.codex/skills/seedance/scripts/seedance.py \
  --prompt-file "<Part提示词.txt>" \
  --output "<输出Part.mp4>" \
  --model ep-20260521101914-nwv8j \
  --ratio 9:16 \
  --duration 15 \
  --resolution 720p \
  --generate-audio \
  --poll-interval 10 \
  --max-wait 5400
```

`env -i` 用来避开本机代理环境污染，例如 `httpx.InvalidURL: Invalid port ':1'`。

## 生成后验证和拼接

检查音视频流：

```bash
ffprobe -v error \
  -show_entries stream=index,codec_type,codec_name,width,height,duration \
  -show_entries format=duration \
  -of json "<输出Part或最终视频.mp4>"
```

拼接两个Part：

```bash
ffmpeg -y -f concat -safe 0 -i <(printf "file '%s'\nfile '%s'\n" \
  "<Part1.mp4>" "<Part2.mp4>") \
  -c copy "<最终30秒.mp4>"
```

打包交付：

```bash
zip -j -9 "<交付包.zip>" "<最终视频1.mp4>" "<最终视频2.mp4>" "<静音保险版.mp4>" "<预览图.jpg>"
```

## 分段策略

| 视频总时长 | 分段方式 | Seedance 输入 |
|-----------|---------|--------------|
| ≤15秒 | 1段 | 全部改图 |
| 16-30秒 | 2段（各15秒） | Part1 改图 + Part2 改图 |
| 31-40秒 | 3段（15秒 + 15秒 + 尾段） | Part1/2/3 改图；尾段按剩余口播长度，可加0.3-1.0秒容错 |
| >40秒 | 默认放弃或先请用户确认 | 不直接批量生成 |

如果用户要求“缩到30秒”，无论原片是90秒还是4分钟，都按两个15秒Part压缩整体剧情，不走上表的全长拆分。

## 后期拼接

多段 Seedance 视频生成后，用 ffmpeg 拼接：

```bash
echo "file 'part1.mp4'" > list.txt
echo "file 'part2.mp4'" >> list.txt
ffmpeg -f concat -safe 0 -i list.txt -c copy output_final.mp4
```

## 常见问题

### Q: 人物在两段之间不一致怎么办？
不要写“同上”或“同Part1完全一致”。每个 Part 都重复关键人物特征，并在人物设定里 @ 对应身份图一次。

### Q: 产品在画面中不够清晰？
在每个3秒段的"画面："里写清楚产品具体细节，比如"能清晰看到鸭脖的骨骼结构、紧实肉质和表面的椒盐调料颗粒"。

### Q: Seedance 生成的视频保留了分镜网格排版？
在视频要求里加上"成片是正常短视频，不要保留分镜图的黑底排版"。

### Q: 动作不自然？
用极具体的动作描述。不写"展示产品"，写"手持鸭脖举至脸旁，目光看向镜头，另一手指向鸭脖，表情兴奋认真"。

### Q: 接缝处停住不动？
不要写“静止”“停留”“焦点锁”。改写成连续动作桥：合盖、收手、转向水龙头、低头洗脸、毛巾擦脸、水声延续、手部操作声延续。
