# Seedance 2.0 多图引用是否需要反复 `@图片N`

日期：2026-07-16

## 结论

**不需要在后续每个镜头反复 `@图片2`。** 更合适的写法是：

1. 在每个独立 Seedance 请求开头，明确把素材和稳定名称绑定起来，例如：`将@图片2中的白色方圆罐定义为“孔凤春泥膜罐”`。
2. 后续始终使用同一个名称“孔凤春泥膜罐”，不要在“产品 / 它 / 这个 / 白罐 / 泥膜”之间随意切换。
3. 只有在容易混淆、隔了很长一段后重新出现、或者包装/材质必须被强约束的关键镜头，才建议再次写 `@图片2`；这是项目里的稳健性策略，不是官方规定的必填语法。

对 `job-011` 来说，当前 Prompt 在很多连续镜头里重复 `@图片2/@图片3/@图片4`，确实可以精简。精简重点不是简单删掉 `@`，而是先把开头的“素材角色说明”改成**正式命名定义**，再统一使用这些名称。

## 官方一手资料说明了什么

### 1. 已定义主体后，可以只使用固定名称

火山方舟官方《Seedance 2.0 提示词指南》先给出定义句式：

> 将图片1中穿红色连衣裙、戴草帽的女人定义为张红。

随后明确说明：

> 对于已提前定义主体的场景，每次提到主体时，都应使用同一标签。

这表示正式定义完成以后，后文可以持续写“张红”，不要求每次都再写 `张红@图片1`。相反，如果没有提前定义，官方才建议每次使用 `<主体N>@<图片N>` 来明确素材绑定。

来源：[火山方舟：Doubao Seedance 2.0 系列提示词指南](https://www.volcengine.com/docs/82379/2222480)

### 2. 官方案例只在开头绑定图片，后面直接使用“博主”和“面霜”

火山方舟官方 SDK 教程给出的美妆案例，开头先写“图片1中美妆博主”与“图片2中的面霜”；后面的动作连续使用“博主”和“面霜”，没有在展示、蘸取、涂脸的每一个动作中重复“图片1/图片2”。

因此，**开头清楚绑定 + 后文稳定共指**是官方实际使用的写法，不只是理论上的可行写法。

来源：[火山方舟：Seedance 2.0 SDK 示例官方教程](https://www.volcengine.com/docs/82379/2291680)

### 3. 上传素材和提示词指代是两层不同的事情

官方 API 示例把图片作为独立 `content` 项上传，并设置 `role: reference_image`。图片的 `图片1 / 图片2` 编号来自它在同类素材中的请求顺序。Prompt 再通过“图片1中的博主”“图片2中的面霜”说明每张素材负责什么。

官方还特别说明：Asset ID 只是把素材传给模型，Prompt 仍应使用“素材类型 + 序号”引用素材，不能拿 Asset ID 当作语义名称。

所以要区分：

- **上传/Asset ID**：保证图片真正进入请求。
- **`@图片N` 或 `图片N`**：在 Prompt 中完成第一次角色绑定。
- **“孔凤春泥膜罐 / 男主播 / 乳白厚泥”**：绑定完成后的自然语言共指。

来源：[火山方舟：Seedance 2.0 SDK 示例官方教程](https://www.volcengine.com/docs/82379/2291680)

## 官方资料没有证明什么

在查到的字节/火山方舟官方资料里，没有找到以下结论：

- “每个镜头都必须重新 `@图片N`”；
- “重复 `@图片N` 的次数越多，参考权重越高”；
- “每次出现产品都重新 `@图片2`，一致性会有可量化提升”。

官方要求的是**主体明确、标签稳定、不要省略到产生歧义**。因此不能把重复 `@` 当成官方规定，也不能声称它一定能提升权重。

## 本项目的实测证据

### 开头绑定、后面不重复，也能识别产品和人物

`job-001` 的一次真实 Seedance 2.0 测试在开头绑定了 `@图片2` 产品和 `@图片4` 人物，后面的镜头主要直接写“孔凤春发酵水”“女主”，没有持续重复 `@`。视觉检查确认目标瓶身和人物身份都贯穿了视频。该测试另有字幕问题，但不是素材共指问题。

### 重复引用有效，但不能证明“必须重复”

`job-011` 当前验证结果使用了较多定点 `@图片N`，普通 Seedance 2.0 一次生成成功，没有 Seedance 重试。但项目没有做“相同 Prompt 只改变 `@图片N` 重复次数”的受控 A/B，因此只能证明重复写法可用，不能证明它是必要条件或优于命名后共指。

## 对 job-011 的具体建议

### 开头改成正式定义

每个 Part 都是一次独立生成，所以每个 Part 的开头都要重新定义：

```text
将@图片1中带 Shot 01–12 编号的连续分镜定义为“分镜板”，只控制镜头顺序、场景、景别、动作和硬切节奏。
将@图片2中的白色方圆罐定义为“孔凤春泥膜罐”，只校准包装、比例和原生标签。
将@图片3中开罐后的乳白色浓稠膏体定义为“乳白厚泥”，只校准泥体颜色、纹理和堆叠状态。
将@图片4中的黑发、深色短袖男性定义为“男主播”，全片涉及主角时始终使用“男主播”。
@图片5只校准同一男主播洗净后的肤况，不定义第二个人物。
```

### 后文怎么写

| 素材 | 后续常规写法 | 建议再次写 `@图片N` 的位置 |
|---|---|---|
| `@图片1` 分镜板 | 直接使用 `Shot 01–02` 和“按分镜板顺序硬切” | 通常不需要每个时间块都写；每个 Part 开头定义一次即可 |
| `@图片2` 产品正面 | 始终写“孔凤春泥膜罐” | 产品第一次正式出现；最终标签特写可再锚一次。连续产品镜头不必句句 `@图片2` |
| `@图片3` 开罐厚泥 | 始终写“乳白厚泥” | 第一次开罐取泥；Part 2 的见底罐内部材质镜头可再锚一次 |
| `@图片4` 男主播 | 始终写“同一男主播”或“男主播” | 开头定义后通常无需重复；若实测发生换脸，再在长时间离开人物后的重新入镜处定点重提 |
| `@图片5` 洗后状态 | 始终写“同一男主播的洗后肤况” | Part 2 第一次进入洗净后近景时写一次；后续无需重复 |

### 推荐的产品写法

不建议：

```text
把@图片2白罐举起，随后把@图片2白罐靠近镜头，再把@图片2白罐转向正面。
```

建议：

```text
将@图片2中的产品定义为“孔凤春泥膜罐”。
男主播举起孔凤春泥膜罐，随后将泥膜罐靠近镜头并转到正面，原生标签清楚。
```

最终收口如果希望额外加强包装事实，可以定点写一次：

```text
男主播把孔凤春泥膜罐贴近镜头，包装和原生标签再次按@图片2校准。
```

## 最终判断

对当前项目，最佳规则不是“永远只在开头写一次”，也不是“每个镜头都反复艾特”，而是：

> **每个独立 Part 开头正式命名；后文始终使用同一个名称；只在第一次出现、状态切换或关键产品特写处定点重提参考图。**

其中“正式命名后使用同一标签”来自官方规则；“关键镜头定点重提”是本项目在缺少受控 A/B 时采用的保守工程策略。
