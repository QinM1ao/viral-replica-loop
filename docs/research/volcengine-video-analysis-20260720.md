# 火山引擎优化视频分析环节：第一性原理审视

- 日期：2026-07-20
- 范围：官方公开资料 + 1 条 25.54 秒样片的 AI MediaKit ASR 真实付费对比。
- 用户入口：[AI MediaKit 文档库 6448](https://docs.volcengine.com/docs/6448/?lang=zh)

## 结论

**值得做真实 A/B，但不应直接把现有视频分析整段替换成火山方案。**

最有价值的变化是把火山豆包 BigASR 加入“听清原话”层：它公开支持字/词时间戳、标点、ITN、语义顺滑、VAD 分句、说话人分离、热词和替换词，这些能力比当前本地 `Qwen3-ASR-0.6B` 的简单全文/chunks 输出更适合品牌词、产品词和时间轴证据。

但官方没有 BigASR 对 Qwen3-ASR 的同集对比，不能仅凭宣传断言“火山一定更准”。本次已用一条校正样片实测 MediaKit 基础 ASR；正确决策是再对 BigASR 2.0 旁路 A/B，用同集人工真值决定主备关系。

ASR 只回答“说了什么、何时说”，不能回答“画面发生了什么”。镜头切分、可见字幕、人物动作、产品状态、before/peak/after 物理变化和故事语义仍需独立证据。

## 本次真实对比

同一条 25.54 秒美妆口播视频，以项目中经画面字幕核对的文稿为参考：

| 项目 | Qwen3-ASR-0.6B | AI MediaKit ASR |
|---|---:|---:|
| 归一化字符错误率 | 8.18% (13/159) | 10.06% (16/159) |
| 时间结构 | 整段 1 chunk | 18 段起止时间 + 段级置信度 |
| 产品/成分词 | `壬二酸` 正确，`白云山` 错 | `白云山` 正确，`壬二酸` 错为 `任二孙` |
| 耗时 | 旧版并行分析步骤 11.949s | 约 14s（服务端 12s） |
| 本机 MLX 负载 | 有 | 无 |
| 本次估算费用 | 本地计算 | 约 0.01277 元 |

这只是一条样片，但已能否定“MediaKit 基础 ASR 肯定更准”。它赢在时间轴和本机资源，没赢在这条样片的文字准确率。详细错误对照、任务 ID 和脱敏原始响应见 `output/job-011/experiments/volcengine_asr_comparison_20260720/`。

当前环境已有 MediaKit API Key，但未配置豆包语音 BigASR 的 Key，因此本次无法实测 `volc.seedasr.auc`。

## 当前流程的真正薄弱点

1. `tools/asr_transcribe.py` 的默认 context 写死了清洁泥膜、壬二酸等词。它会帮助本次泥膜样片，但对化妆水、其他 SKU 或无关类目会形成错误先验。无论最后用 Qwen 还是 BigASR，context/热词都应该从当前 `product_profile.json`、源文件名和可见字幕动态构建。
2. 当前 Qwen 样片只输出覆盖全片的 1 个 chunk；`source_rhythm` 后续只注入全文，beat 与口播用字符位置绑定，不是原生语音时间轴。BigASR 2.0 的 utterance/word timestamps 可以直接改善这一层证据。
3. 新版 `source_blueprint` 已将 ASR、联系表、Seed 2.0 Mini 全片 2fps 和开头 5fps 理解并行。本次已有报告中，视频语义分析关键路径是 57.617s，本地切点/证据帧只是 3.012s。因此换 ASR 不会显著缩短当前总耗时，主要收益是文本和时间证据质量、以及释放本机 MLX 内存。

## 先分清两条产品路线

### AI MediaKit 6448：易接视频，但公开 ASR 结果较薄

[MediaKit ASR API](https://docs.volcengine.com/docs/6448/2386124?lang=zh)可直接输入视频/音频 URL，异步提交后轮询或回调；公开支持常见视频与音频格式、最长 3 小时、中文/英文自动探测、可选说话人和置信度。

公开响应核心是句级 `subtitles[]`：字幕文本、开始时间、结束时间，以及可选 speaker/confidence。文档没有公开 word-level timestamp、热词或请求级上下文字段。

因此它适合“快速把视频变成可读字幕”，但不一定适合作为当前 `source_rhythm` 的精确词级证据主源。

### 豆包语音 6561 BigASR：更适合精确语音证据

[语音识别大模型产品简介](https://www.volcengine.com/docs/6561/1354871?lang=zh)公开的相关能力包括：

- 字/词与分句时间戳、停顿信息
- 自动标点、数字规整 ITN、语义顺滑 DDC、VAD 分句
- 中英文说话人分离、双声道识别
- 中英文、国内多种方言/口音及部分外语
- 热词、请求级上下文、正则/术语替换
- 语种、性别、语速、音量、情绪等扩展检测（具体端点能力不同）

三种录音文件接口中：

| 路线 | 调用方式 | 适配判断 |
|---|---|---|
| [标准版](https://www.volcengine.com/docs/6561/1354868?lang=zh) | 异步提交/查询，可回调；文档列出模型 1.0 `volc.bigasr.auc` 和 2.0 `volc.seedasr.auc` | 功能最全，适合质量 A/B；需要可下载音频 URL |
| [极速版](https://www.volcengine.com/docs/6561/1631584?lang=zh) | 单次同步；最长 2 小时、100 MB，可 URL 或 Base64 | 最适合短视频 MVP；省去轮询，但扩展检测能力少于标准版 |
| [闲时版](https://www.volcengine.com/docs/6561/1840838?lang=zh) | 异步，24 小时内 | 不适合当前非图片路径 20 分钟预算 |

为公平测试，应先把所有输入统一抽成 16 kHz mono WAV，避免容器和解码差异污染 ASR 对比。

## 火山能力应放在哪一层

```text
BigASR
  原话、品牌/成分词、词级时间、停顿、说话人候选

OCR / 可见文字
  画内字幕、产品标签、模型错词交叉校正

实测切点 + 5fps 帧
  硬切、动作峰值、before/peak/after 物理状态

Seed 2.0 Mini
  故事语义、角色、镜头功能、不确定项

checker
  冲突裁决；以直接音频、像素和实测时间证据优先
```

官方将[场景切分](https://docs.volcengine.com/docs/6448/2386118?lang=zh)、[智能语义切片](https://docs.volcengine.com/docs/6448/2563057?lang=zh)、[视频理解智能策略](https://docs.volcengine.com/docs/6448/2552748?lang=zh)和 ASR 拆成独立工具，本身就说明 ASR 不是视觉分析或场景切分的替代品。MediaKit ASR 响应也没有帧、镜头、动作或物理状态字段。

## 当前 6448 计费口径

按 [AI MediaKit 当前计费说明](https://docs.volcengine.com/docs/6448/2486473?lang=zh)：

| 能力 | 单价 |
|---|---:|
| ASR | 0.03 元/分钟 |
| OCR | 0.25 元/分钟 |
| 场景切分 | 0.02 元/分钟 |
| 语义切片 | 0.03 元/分钟 |

这意味着 MediaKit ASR 折合 1.8 元/小时；OCR 成本明显更高，应优先用在关键帧或争议片段，而不是无差别全量调用。场景切分和语义切片可作为现有实测 cut / Seed 语义层的对照候选，不能未验证就叠加为默认成本。

旧 VOD `/docs/4` 页面中的历史价格不是用户本次 6448 MediaKit 入口的计费口径，本报告不采用。

[豆包产品页](https://www.volcengine.com/product/doubao)本次公开页面列出“Doubao-录音文件识别2.0”0.8 元/小时，并列录音文件识别 20 小时免费额度；实际开通时仍应在控制台确认免费额度绑定的版本、资源 ID 和有效期。

## 最小真实 A/B

先不改 runner，只旁路测试 10–20 条已有视频：

1. 样本覆盖干净单人口播、音乐/噪声口播、多人或画外音；至少一条含品牌/成分词和中英混说。
2. 同一抽取音轨并跑：当前 Qwen3-ASR、MediaKit 基础 ASR、BigASR 标准版 2.0；极速版用于补测延时，不与 2.0 质量等同。
3. 保存火山原始响应、请求参数、logid、word timestamps、speaker；原始文本与 ITN/DDC 后文本分开，避免后处理改写原话而不可审计。
4. 人工真值评分：
   - 品牌/产品/成分关键词准确率
   - 整体字错率或人工编辑距离
   - 漏句、幻觉句
   - 句级/词级边界误差
   - 说话人切换正确率
   - 端到端耗时、失败/重试和单条费用
5. 只有在关键词不退步、整体错误稳定下降、时间戳可支撑 `source_rhythm`、P95 延时不破坏 20 分钟预算时，才把 BigASR 升为主证据；否则保留 Qwen 主路径，用火山复核争议词。

推荐顺序：先用标准版 2.0 验证质量，因为当前关键路径由约 57.6s 的视频语义分析主导，ASR 已并行，极速版的延时优势暂时不会缩短总耗时。闲时版不进入快路径。

## 风险与边界

- BigASR 标准版需要服务端可下载 URL，极速版可传 URL/Base64；MediaKit 接受公网及火山云媒资协议。音频都会离开本机。
- 当前录音文件 HTTP 文档未明确给出输入保留时长、训练使用、删除 SLA 和可选地域清单；敏感人声上线前需通过合同或工单确认。
- DDC 会顺滑口语，ITN 会规范数字；二者都可能改变逐字原文，必须保留 raw 与 normalized 两层。
- 热词适合“孔凤春、壬二酸”等词，但过多常用词可能伤害整体识别，应按任务小词表测试。
- 官方准确率宣称是相对传统模型，不是相对当前 Qwen3-ASR；最终结论必须来自同集实测。

## 落地建议

1. 保留现有 Seed 2.0 Mini、实测切点、5fps 像素证据和 checker。
2. 将 BigASR 作为旁路 exact-words 候选，先不动状态机和门禁。
3. A/B 通过后，把火山原始响应保存为一等证据，Qwen 留作离线 fallback。
4. MediaKit 场景切分、语义切片和 OCR 分别做小样对照；只有证明增加了独立信息，才进入默认流程。

一句话：**火山最可能明显提升的是“语音转写的精度与时间结构”，不是整个视频理解；先替换一层证据，实测通过后再扩大，而不是把视觉分析交给 ASR。**
