# 火山引擎：成片仅「洗后」片段美颜的可调用能力调研

日期：2026-07-21。范围：只评估已经生成好的视频，能否只处理洗后片段；未提交任何付费任务。

## 结论

目前没有查到一个可直接把 MP4 上传、指定 `start/end`、只给人脸轻美颜并返回成片的火山引擎云端 API。最接近「真正视频美颜」的是智能美化特效 SDK：它对本地解码出的每帧做实时处理，需要我们自己控制时间段并重新编码，不是 MediaKit 式的服务端成片任务。[智能美化特效 SDK 概览](https://www.volcengine.com/docs/6705/1162076?lang=zh) [iOS 本地视频处理](https://www.volcengine.com/docs/6705/101961)

对本仓库，暂不建议把美颜接入默认收尾链路。洗后镜头若本身不达标，优先回到生成阶段重做该镜头；这是对人物身份、皮肤质感和产品可见文字最可控的路径。若必须修已有成片，先取得 SDK 试用/授权，再做一支短的、无产品标签的洗后片段验证；不要直接全片处理。

## 能力对比

| 方案 | 已有视频 + 指定洗后区间 | 服务端成片输出 | 鉴权/计费 | 适配判断 |
| --- | --- | --- | --- | --- |
| AI 美颜 API（`Live`） | 否。`resource_list` 文档明确为图片 URL 或 data URI，不接收视频；因此也没有 MP4 时间段语义。 | 无视频成片合同。 | 需向商务申请 Access Key，并按签名协议调用。 | 不能作为成片美颜；最多可研究静帧效果/参数。 |
| 智能美化特效 SDK | 可以作为工程实现：本地视频由 `AVAssetReader` 解成带时间戳的帧，应用层只在洗后区间开启效果。SDK 文档没有承诺直接导出 MP4，需自行编码、封装和保留原音频。 | 否，原生端/PC 的帧处理 SDK。 | 商业 SDK 许可证；具体套餐需商务确认，不是按一次 MP4 任务调用。 | 唯一匹配「轻美颜」的技术路径，但接入成本和回归质检都高。 |
| RTC 基础/智能美颜 | 否。面向本地采集的视频通话/直播流；Web 文档还限定基础美颜仅本地主流，不支持屏幕流或远端流。把文件伪装成流再录制是非标准绕行。 | 否，运行时渲染效果。 | RTC AppID/Token；基础美颜免费，智能特效另需 CV 授权。 | 不适合离线后期。 |
| AI MediaKit「画质增强（大模型）」 | 可提交整段已有视频，但请求体没有 `start/end`、人脸或局部蒙版参数。若使用，只能先本地切出洗后段、云端处理后再拼回去。 | 是，异步任务返回视频；可写 VOD/TOS，临时 HTTPS 结果保留 24 小时。 | `Bearer MediaKit API Key`；按实际控制台产品计费与权限执行。 | 这是画质修复/超分，不是美颜；扩散式生成会改动全画面，不应拿来解决肤质问题。 |

### 1. 云端美颜：现有公开 API 不满足视频后期

[AI 美颜 Live API](https://www.volcengine.com/docs/6705/2441391?lang=zh) 虽名为「Live」，输入规格仍是图片，模板为 `beautyLive`；它不构成视频美颜 API。其 Pro 页面公开可见但未提供可验证的视频输入、时间范围和成片输出合同，不能据名称推断具备视频能力。[AI 美颜 Pro](https://www.volcengine.com/docs/6705/2441392?lang=zh)

已检查的 [AI MediaKit 视频工具文档](https://www.volcengine.com/docs/6448) 中未找到「视频人脸美颜」的公开任务接口。MediaKit 的生成式画质增强接口是 `POST /api/v1/tools/enhance-video-generative`，接受完整 `video_url`，输出仅支持 720p/1080p，且没有片段或人脸区域字段。[画质增强（大模型）API](https://www.volcengine.com/docs/6448/2464595?lang=zh) 因此「先切洗后段再处理」是基于接口参数缺失作出的工程推断，不是该 API 的原生时间轴能力。

这类增强会以生成式方式补全压缩、老化丢失的细节，适合分辨率/噪点类问题，不是皮肤磨皮、提亮或五官微调。若画面内或转场邻近产品、瓶身文字，生成式重建有篡改文字和身份细节的风险；即使短片段测试通过，也必须逐帧检查。

### 2. 智能美化特效 SDK：可做，但要自己完成离线后期

SDK 支持磨皮、美白、锐化、清晰和人脸塑形等视频特效能力。[美颜能力说明](https://www.volcengine.com/docs/6705/1521678?lang=zh) iOS 的本地视频模式以 `AVAssetReader` 解码本地视频并把帧送入处理回调；这使「仅洗后时段开启轻美颜」可由时间戳控制，但导出视频、音画同步和回拼是我们的责任，而不是该 SDK 的一个云端任务。[本地视频处理说明](https://www.volcengine.com/docs/6705/101961)

若验证，建议只启用低强度的磨皮/清晰或美白，禁止瘦脸、改脸型和大幅五官重塑；并在处理前后做洗后参考帧的人物一致性、皮肤自然度、字幕/产品文字可读性检查。SDK 的人脸检测和整帧渲染不是「保护产品标签」的内容保真承诺，标签、眼镜、发丝和边缘都要作为风险点验收。

### 3. RTC 美颜：适合互动，不是成片加工

[RTC 基础美颜](https://www.volcengine.com/docs/6348/1134358) 是本地采集和预览中的实时效果；[Web 限制](https://www.volcengine.com/docs/6348/148646) 明确其只作用于本地主流。智能美颜则通过客户端 CV 资源和效果节点启用。[RTC 智能美颜效果](https://www.volcengine.com/docs/6348/152576) 它们不接受一个已有 MP4 后返回替换 MP4，也没有洗后时间段任务接口，故不纳入本项目的离线修片方案。

## 建议与下一步

1. 不在当前默认收尾流程中接入美颜；维持「生成失败则重做洗后镜头」的策略。
2. 若问题实为压缩、分辨率或噪点而非肤质，可在用户明确授权成本后，用一段独立、无标签的洗后片段试跑一次 MediaKit 画质增强，再本地拼回并做逐帧验收；该验证不能证明美颜能力。
3. 若业务明确要求稳定的洗后轻美颜，先向火山引擎申请智能美化特效 SDK 的试用/报价和离线导出支持确认，再做一支 3--5 秒、无产品的 POC。POC 通过条件应包括：仅影响目标区间、原音频和时长不变、人物不失真、无文字/边缘变形。该 POC 可能涉及商业授权，需另行成本批准；本次调研未发起。
