# Hash-Gated Visual QC

We will reuse heavy visual QC by active image hash after image batch PASS. The loop still blocks true visual failures and missing evidence, but downstream prompt/request stages should not rerun geometry, continuity, skincare progression, mud review, or GPT Image contract QC when the final Part image hashes and approved visual manifest mapping are unchanged; they run lightweight sync checks instead. Non-material visual warnings continue automatically when the review records `why_not_fail`, and image sample review is internal unless the user explicitly asks for it. This trades a small risk of stale visual assumptions for a much faster loop, while keeping safety around changed images, changed mappings, changed reference roles, user-reported visible defects, and hard failures.
